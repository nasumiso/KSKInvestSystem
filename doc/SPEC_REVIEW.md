# 投資システム評価レビュー

> 2026-02-23 Claude Code による評価。コーディング品質ではなく、投資システムとしての有用度を評価。

## 方法論の分類

**CANSLIM + ミネルヴィニSEPA の日本株ローカライズ版**

マルチファクターモデルによる成長株スクリーニング・ランキングシステム。

### スコアリング構成

| ファクター | ウェイト | 主な評価軸 |
|-----------|---------|-----------|
| 業績（gyoseki） | 40% | 5年安定性、年度/四半期利益成長、売上加速度 |
| モメンタム（momentum） | 25% | RS（13/26/39/52週加重）のTOPIX相対正規化 |
| 指標（shihyo） | 20% | 時価総額、自己資本比率、理論株価乖離 |
| ファンダメンタルズ（funda） | 15% | テーマランキング（市場の資金フロー追随） |

---

## 強み

### 1. マルチファクター設計が合理的

- 業績40% + モメンタム25% + 指標20% + テーマ15% のウェイト配分は、成長株投資の王道
- 単一指標への依存を避けており、偏った銘柄選定が起きにくい

### 2. 業績評価の多層構造

- 5年安定性 → 直近年度 → 直近四半期 → 3四半期加速度、と時間軸を重ねて評価
- 「SaaS 40%ルール」の適用により、赤字成長企業も正当に評価できる
- 売上加速ボーナス（直近四半期 > 前四半期で+1/3）は「変化率の変化率」を捉えており、機関投資家的な発想

#### 業績スコア内訳（5要素）

| 要素 | 配点 | 内容 |
|------|------|------|
| 過去5年業績安定性 | 20点 | 利益成長率（年平均7%+が理想）、売上CAGR（10%+が理想）、赤字年は-3点 |
| 直近年度利益成長 | 10点 | 前年比20%+で満点、10%+で半分。SaaS 40%ルール適用 |
| 直近四半期利益成長 | 15点 | 前年同期比40%+で満点、20%+で半分 |
| 3四半期売上成長 | 30点 | 直近3四半期平均。加速ボーナスあり |
| 3四半期利益成長 | 25点 | 営業利益・経常利益・最終利益の3四半期平均。加速ボーナスあり |

### 3. 理論株価モデルが独自

- **解散価値**: BPS × 自己資本比率ディスカウント（50%-80%）
- **事業価値**: EPS × ROA × 150 × 財務補正係数
- **理論株価**: (解散価値 + 事業価値) × リスク補正
- 上限・下限バンドと先行理論株価（来期予想ベース）があり、「いくらまでなら買えるか」の判断材料になる

### 4. 売買シグナルが実用的

| シグナル | 意味 | 用途 |
|---------|------|------|
| 新 | 新高値 | ブレイクアウト検出 |
| 出 | 出来高急増 | 機関投資家の集積 |
| ポ | ポケットピボット（7日以内） | セットアップ形成中 |
| ブ | ブレイクアウト（7日以内） | 確認シグナル |
| 買過 | 売り圧力レシオ >75 | 過熱警告 |
| 売過 | 売り圧力レシオ <25 | 割安圏 |
| 警 | 弱含み形成中 | 撤退検討 |
| 売 | RS弱 + レシオ<45 | 売りシグナル |
| 急 | 1日でランク30%+上昇 | 急騰モメンタム |
| 昇 | 5日でランク30%+上昇 | 持続的モメンタム |

### 5. テーマローテーション追随

- 日本株市場はテーマ相場の比重が大きいため、テーマランキング（上位30テーマ）の組み込みは実践的
- 1日のランク変動をモメンタムシグナルとして反映

### 6. テクニカル指標の網羅性

- **トレンドテンプレート**: MA関係・52週ポジション・RSしきい値の7条件チェック（ミネルヴィニ式）
- **ポケットピボット**: MA付近での出来高確認による集積フェーズ検出
- **売り圧力レシオ**: OHLCからの日中売買圧力推定（5日/20日）
- **RS（相対力指数）**: 13/26/39/52週の加重比較、TOPIX相対の正規分布正規化

## 弱み・改善余地

### 1. ベアマーケット対応が薄い

- RS（相対力）ベースのため、下落相場では全銘柄のスコアが低下してキャッシュ判断を出せない
- 「市場全体が弱い → ポジションを減らす」というマーケットタイミング機能がない
- **改善案**: TOPIX RSが一定以下なら全体エクスポージャーを下げるルールの追加

### 2. バックテストがない（最大の弱点）

- スコアリングで選んだ銘柄が**実際にどれだけリターンを出したか**の検証がない
- ウェイト（40/20/25/15）やしきい値（四半期利益成長40%+で満点、等）が経験則ベースで、統計的根拠が不明
- **改善案**: 過去のランキング上位銘柄のN日後リターンを追跡する仕組みの構築

### 3. リスク管理がシステムに組み込まれていない

- ポジションサイジング、損切りルール、ポートフォリオ分散の仕組みがない
- 「何を買うか」は精緻だが「いくら買うか」「いつ切るか」はユーザー判断に委ねられている
- **改善案**: ATRベースのポジションサイジング、トレーリングストップの導入

### 4. 理論株価モデルの定数が固定的

- `EPS × ROA × 150` の「150」や自己資本割引率の根拠が不透明
- セクター別に適正倍率は異なるはず（IT vs 製造業 vs 金融）
- **改善案**: セクター別の倍率テーブル、または回帰分析による動的倍率

### 5. 出来高分析の限界

- 売り圧力レシオはOHLCからの推定であり、実際の約定データ（板情報）ではない
- 機関投資家の動向把握としては精度に限界がある

## 市場環境別の有効性

| 市場環境 | 有効性 | 理由 |
|---------|--------|------|
| 強気相場 | 高い | RS上位銘柄 + 業績加速 + テーマ追随が噛み合う |
| レンジ相場 | 中程度 | テーマローテーションで一部機能するが、全体スコアは低迷 |
| 弱気相場 | 低い | RS指標が機能しにくく、キャッシュ判断機能がない |

---

## ポジショニング（個人投資家の中での位置づけ）

| レベル | 推定比率 | やっていること |
|--------|---------|---------------|
| 大多数 | 80% | 証券会社のスクリーナー、四季報、SNS情報 |
| 中級 | 15% | Excel/Googleスプレッドシートで独自指標管理 |
| 上級 | 4% | Pythonでスクレイピング + 簡易スコアリング |
| **本システム** | **<1%** | **マルチファクターモデル + 独自理論株価 + テーマ追随 + 自動パイプライン** |
| ヘッジファンド級 | — | バックテスト + リアルタイムAPI + 自動売買 + リスク管理統合 |

**個人投資家の中ではトップ1〜2%** の作り込みレベル。「自動スクリーニング + 手動売買」の形態としては最高峰に近い。

## 次のステップ（優先順位順）

### 1. バックテスト基盤の構築（優先度: 最高）

過去のランキング上位銘柄の実リターンを検証し、スコアリングの有効性を統計的に裏付ける。ウェイトやしきい値の最適化にも繋がる。

### 2. マーケットレジーム判定（優先度: 高）

TOPIX/日経平均のトレンド状態（強気/弱気/レンジ）を自動判定し、弱気相場でのエクスポージャー抑制を実現する。

### 3. ポジション管理モジュール（優先度: 中）

ATRベースのポジションサイジング、損切り/利確ルールをシステムに組み込み、「半自動トレーディングシステム」への進化を図る。
